Questão de Avaliação

Inteligência Artificial no Processo de Programação

Durante o desenvolvimento de um e-commerce, um programador implementou a seguinte função para calcular o total do carrinho:

function calcularTotal(itens) {
  let total = 0;
  for (let i = 0; i <= itens.length; i++) {
    total += itens[i].preco;
  }
  return total;
}

Assinale a alternativa correta considerando o erro no código e o apoio da IA:

A) A Inteligência Artificial pode substituir completamente o programador nesse cenário.
B) O erro ocorre porque a variável total não foi inicializada corretamente.
C) O erro ocorre porque a condição do laço permite acessar uma posição inexistente (undefined), e ferramentas de IA auxiliam na identificação do erro.
D) O erro ocorre porque a função deveria utilizar obrigatoriamente programação orientada a objetos.
E) O problema está relacionado estritamente ao tipo da variável itens.

Correção Comentada

A alternativa correta é a C.

Análise Técnica: O erro está em i <= itens.length. Em JavaScript, arrays começam no índice 0. Se um array tem 3 itens, os índices são 0, 1 e 2. O length será 3. Ao tentar acessar o índice 3 (quando i é igual ao length), o sistema retorna undefined, causando um erro ao tentar somar com o total.

O papel da IA: Ferramentas como Copilot ou ChatGPT identificam esse padrão instantaneamente, sugerindo a correção para i < itens.length ou o uso de .reduce(). Contudo, cabe ao desenvolvedor validar se a sugestão da IA não ignora regras de negócio específicas (como taxas ou descontos).
